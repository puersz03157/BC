【已在本機做好的事】
- 已複製一份與「荒野：家園.html」相同內容的 index.html。Pages 預設只會自動開 index.html，沒有這個檔時根網址會 404。
- 已保留 .nojekyll（避免 Jekyll 處理壞檔）。
- 已在 wilderness_home 資料夾內 git init，並完成第一次 commit（含專案原始檔與 docs 網頁包）。

【你在 GitHub 上要做的】
1. 在 github.com 建立新倉庫（不要勾選自動加 README，避免 push 衝突），記下倉庫網址，例如 https://github.com/你的帳號/wilderness_home.git
2. 在本機 PowerShell 執行（路徑與網址改成你的）：

   cd "C:\Users\puers\Desktop\CB\wilderness_home"
   git remote add origin https://github.com/你的帳號/倉庫名稱.git
   git push -u origin main

3. 倉庫 → Settings → Pages → Build and deployment → Deploy from a branch → Branch 選 main → Folder 選 /docs → Save。
4. 約 1～3 分鐘後開：https://你的帳號.github.io/倉庫名稱/

【之後更新網頁版】
在 Godot 重新 Web 匯出後，把新檔覆蓋進 docs/，再執行 git add docs、git commit、git push。
【重要】根網址 https://帳號.github.io/倉庫名/ 只會讀 index.html。匯出後請將「荒野：家園.html」複製並覆蓋成 index.html（與首次上架說明相同），否則 GitHub 上已更新檔案，頁面仍會看起來是舊的。

【若已更新仍像舊版】
1) 確認 Godot 已「重新匯出」Web，且 docs 內 .pck 的檔案大小有變（或至少與 index.html 裡 GODOT_CONFIG 的 fileSizes 一致）。程式在 .pck 裡，只改 .gd 沒重匯＝線上永遠舊版。
2) index.html 已內建「以 .pck 位元組數當 query」避免 GitHub Pages 快取舊包；若仍異常，請用 Chrome 開站台 → F12 → Application → Clear site data，再強制重新整理（Ctrl+F5）。

【倉庫根目錄若在上一層】
若 Git 倉庫是整個 CB 而不是 wilderness_home，請把網頁匯出放在「倉庫根」的 docs/，並在該層 git；Pages 只認倉庫最外層的 /docs。
